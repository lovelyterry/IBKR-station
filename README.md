# IBKR 交易记录解析工具 (`prase.py`)

> 将盈透证券（IBKR）导出的 CSV 交易记录一键导入内存，并在终端中进行交互式分析。

---

## 功能概览

`prase.py` 是一个纯 Python 命令行工具，专为解析和分析 **Interactive Brokers（IBKR）** 导出的交易流水 CSV 文件而设计。它能够：

- **自动识别** IBKR 多段式（Header/Data）CSV 格式，也兼容普通标准 CSV
- **按证券代码分组**所有交易记录（股票 + 关联期权自动归并至同一标的）
- **提供交互式命令行终端**，支持实时查询、筛选、汇总，以及进入 Python Shell 进行自定义分析
- **计算持仓成本与盈亏**，采用加权平均成本法，支持多头/空头/仓位反转，并单独核算期权权利金

## 快速开始

将 IBKR 导出的 CSV 文件（如 `U13968552.TRANSACTIONS.*.csv`）放入同一目录，然后运行：

```bash
# 扫描当前目录下所有 CSV 文件
python prase.py

# 指定目录
python prase.py --dir /path/to/csv/folder

# 禁用自动类型转换（保留原始字符串格式）
python prase.py --no-convert
```

## 交互式终端命令

启动后进入 `(pandas-db)` 交互终端，支持以下命令：

| 命令 | 格式 | 说明 |
|------|------|------|
| `code` | `code <代码>` | 查看指定标的的全部交易记录及当前持仓成本 |
| `summary` | `summary` | 查看所有标的的持仓汇总与账户盈亏总览 |
| `query` | `query <表名> <表达式>` | 使用 Pandas 表达式筛选数据 |
| `sum` | `sum <表名> <列名>` | 计算指定列的数值总和 |
| `save` | `save <表名> <文件路径>` | 将指定表导出为 CSV 文件 |
| `shell` | `shell` | 进入原生 Python Shell，直接操作 DataFrame |
| `exit` / `quit` / `q` | — | 退出程序 |

### 示例

```
# 查看 NVO 的所有交易明细与持仓成本
(pandas-db) code NVO

# 按条件筛选
(pandas-db) query "Transaction History" "数量 > 100"
(pandas-db) query "Transaction History" "代码 == 'SGOV'"

# 查看账户整体盈亏总结
(pandas-db) summary

# 导出某张表
(pandas-db) save "Transaction History" output.csv
```

## 核心功能详解

### 1. CSV 解析

- **IBKR 多段式 CSV**：自动按 `Header/Data` 行拆分为多个独立 DataFrame（如 `Transaction History`、`Cash Report` 等），并将数据行从倒序还原为正序
- **标准 CSV**：直接加载，以文件名（去除扩展名）作为表名

### 2. 字段标准化

自动将英文表头和交易类型映射为中文：

| 原始英文 | 映射中文 |
|----------|----------|
| Date | 日期 |
| Symbol | 代码 |
| Quantity | 数量 |
| Price | 价格 |
| Amount | 总额 |
| Commission | 佣金 |
| Net | 净额 |
| Buy / Sell | 买 / 卖 |
| Dividend | 股息 |
| Withholding Tax | 外国预扣税 |
| … | … |

只保留分析所需的核心字段：`日期、账户、说明、交易类型、代码、数量、价格、总额、佣金、净额`。

### 3. 按代码分组

程序启动后自动将所有含 `代码` 列的表按标的代码分组。期权记录（代码含空格，如 `NVO   260102P00052000`）会自动归并到对应标的（`NVO`）的组内，统一在 `code NVO` 命令下展示。

### 4. 持仓成本计算（`code` 命令）

对每个标的输出：

- **当前净持仓数量**（支持空头负值）
- **总持仓成本** 与 **每股平均成本**（加权平均成本法）
- **全投入均价**（含已实现做T盈亏，反映真实现金投入）
- **含期权的全投入均价**（将期权净权利金计入成本）
- **已实现盈亏**（股票做T差价）
- **期权净权利金**（所有关联期权净现金流合计）

### 5. 账户总览（`summary` 命令）

以表格形式汇总所有代码，分为**持仓中**和**已平仓**两组，并在末尾输出：

- 已实现盈亏（股票）
- 期权净权利金
- 股息净收入（含预扣税、替代支付）
- 利息净收入
- 佣金合计
- 其它费用与调整
- **账户净盈亏合计**
- 资金转账合计（存款/取款）

## 注意事项

- 需要使用 IBKR 账户管理中的"**报告 → 交易流水**"导出 CSV 格式的对账单
- 脚本会将所有 CSV 合并加载，适合将多个时间段的对账单放在同一目录下一次性分析
- 默认会过滤掉不含核心字段的 Section（如 `Statement`、`总结` 等），仅保留有效交易数据
- 如果列名包含空格，在 `query` 命令中请用反引号括起，例如：`` `Price Currency` == 'USD' ``
